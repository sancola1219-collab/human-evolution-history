# 人物頭部被 translate 兩次，害兩顆頭飛到畫面頂端

**症狀**：近看場景裡，人物的身體在地面正常，但頭卻飄在畫面最上方（營火場景出現「兩顆浮頭」）。

**原因**：`drawFigure` 畫頭時先 `ctx.translate(0, headY)`（其實寫成 `headY - hipY + hipY` 等於 headY），接著又 `ctx.translate(hx, headY)` 再位移一次，導致 y 總位移 = `2 × headY`。`headY` 是負值（在腳底之上約 -H），乘二就跑到 `-2H`，遠超出身體，貼到畫面頂。

**正確做法（已採用）**：畫頭**只位移一次**到「肩上一點」：
```js
ctx.save();
ctx.translate(sx, headY);   // 一次到位：sx=肩x, headY=頭心y（相對腳底）
// ...在原點畫顱骨/臉/眉脊/眼...
ctx.restore();
```
並補一段脖子把肩 `(sx, sy)` 連到頭底 `(sx, headY+8s)`，看起來才連貫。

**為什麼重要**：這是「相對座標疊加」最典型的 bug——每個 `translate` 都是**累加**的。畫身體各部位時要清楚「目前原點在哪」，可疑就 `save/restore` 包起來、只做一次位移。像素取樣看不出這種「位置錯但有畫」的錯，一定要**實際看一幀 PNG** 才會發現。
